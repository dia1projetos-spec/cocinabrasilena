1

Criar o Projeto no Firebase

Acesse o console e configure o projeto base

🌐

Acesse o Firebase Console

Abra console.firebase.google.com no seu navegador. Faça login com sua conta Google.

➕

Criar novo projeto

Clique em "Adicionar projeto". Digite o nome: cocina-brasilena (ou qualquer nome). Pode desativar o Google Analytics (não precisamos). Clique em "Criar projeto".

⏳

Aguardar criação

O Firebase cria o projeto em ~30 segundos. Quando aparecer "Seu projeto está pronto", clique em "Continuar".

2

Ativar o Firestore Database

Banco de dados em tempo real para produtos, slides e configurações

🗄️

Ir para Firestore

No menu lateral esquerdo, clique em "Firestore Database". Depois em "Criar banco de dados".

🔒

Modo de início

Selecione "Iniciar no modo de produção". Não se preocupe — vamos configurar as regras corretas no Passo 4.

🌍

Localização

Selecione southamerica-east1 (São Paulo) para menor latência para seus clientes no Brasil. Clique em "Ativar".

ℹ️ O Firestore vai criar automaticamente as coleções quando o admin cadastrar o primeiro produto, slide ou configuração. Não é necessário criar manualmente.

📋 Estrutura das coleções que serão criadas automaticamente:

ColeçãoO que armazenaQuem cria
productsProdutos do cardápio (nome, preço, foto, categoria...)Admin
slidesImagens do banner/slider da página inicialAdmin
config/logoURL do logo da lojaAdmin
config/statusSe está "aberto" ou "cerrado" + mensagemAdmin
3

Ativar Authentication (Login do Admin)

Criar o usuário administrador do painel

🔐

Ir para Authentication

No menu lateral, clique em "Authentication". Depois clique em "Começar".

✉️

Habilitar E-mail / Senha

Clique em "E-mail/senha". Ative o primeiro toggle (E-mail/senha). Clique em "Salvar".

👤

Criar o usuário admin

Vá na aba "Usuários" → "Adicionar usuário". Digite seu e-mail e crie uma senha forte (guarde bem!). Clique em "Adicionar usuário".

⚠️ Importante: Use um e-mail e senha que você não vai esquecer. Esse é o único acesso ao painel administrativo do seu restaurante.
4

Configurar Regras de Segurança

Clientes só leem; admin escreve, edita e deleta

📋

Acessar as Regras

No Firestore Database, clique na aba "Regras" no topo da página.

📝

Substituir pelo código abaixo

Apague tudo que está no editor e cole o código abaixo. Depois clique em "Publicar".

Firestore → Database → Regras — Cole exatamente este código:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Produtos: qualquer um lê, só admin escreve match /products/{productId} { allow read: if true; allow write: if request.auth != null; } // Slides do banner match /slides/{slideId} { allow read: if true; allow write: if request.auth != null; } // Config (logo, status aberto/fechado) match /config/{configId} { allow read: if true; allow write: if request.auth != null; } } }
✅ Essas regras garantem que clientes vejam o cardápio sem precisar de conta, mas somente você (logado no admin) pode adicionar, editar ou deletar produtos.
5

Conectar as Credenciais ao Site

Copiar as chaves do Firebase para o arquivo de configuração

⚙️

Acessar configurações do projeto

Clique no ícone de engrenagem ⚙️ ao lado de "Visão geral do projeto" no menu lateral. Selecione "Configurações do projeto".

🌐

Adicionar app da Web

Role a página até "Seus apps". Clique no ícone </> (Web). Digite um apelido: cocina-site. Não marque "Firebase Hosting". Clique em "Registrar app".

📋

Copiar as credenciais

O Firebase vai mostrar um bloco de código com o firebaseConfig. Copie os valores.

✏️

Editar o arquivo js/firebase-config.js

Abra o arquivo no seu computador e substitua os valores de exemplo pelos reais:

js/firebase-config.js — Substitua com suas credenciais reais:
// ============================================= // Cole aqui as credenciais do seu Firebase // ============================================= import { initializeApp } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-app.js"; import { getFirestore } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-firestore.js"; import { getAuth } from "https://www.gstatic.com/firebasejs/10.12.0/firebase-auth.js"; const firebaseConfig = { apiKey: "AIzaSy...", // ← cole o seu authDomain: "seu-projeto.firebaseapp.com", projectId: "seu-projeto", storageBucket: "seu-projeto.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" }; const app = initializeApp(firebaseConfig); export const db = getFirestore(app); export const auth = getAuth(app); // ============================================= // Cloudinary — preencher no Passo 6 // ============================================= export const CLOUDINARY_CONFIG = { cloudName: "seu-cloud-name", // ← preencher no passo 6 uploadPreset: "seu-preset" // ← preencher no passo 6 };
🔑 Segurança: O apiKey do Firebase para Web é público por design — ele é necessário para o site funcionar. A segurança real está nas Regras do Firestore que configuramos no Passo 4.
6

Configurar o Cloudinary (Imagens)

Hospedagem de fotos dos produtos, logo e slides

📝

Criar conta gratuita

Acesse cloudinary.com → clique em "Sign Up For Free". A conta gratuita oferece 25 GB de armazenamento — mais do que suficiente para começar.

📋

Anotar o Cloud Name

Após o login, no Dashboard você verá o "Cloud name" no topo (ex: dxyz123abc). Anote esse valor.

⚙️

Criar Upload Preset

Vá em Settings (ícone de engrenagem) → Upload → role até "Upload presets" → clique em "Add upload preset".

🔓

Configurar o preset

Em "Signing Mode" selecione "Unsigned" (obrigatório para funcionar pelo navegador sem backend). Anote o "Preset name" gerado. Clique em "Save".

✏️

Atualizar o firebase-config.js

Preencha os valores de Cloudinary no arquivo:

Parte final do firebase-config.js:
export const CLOUDINARY_CONFIG = { cloudName: "dxyz123abc", // ← seu Cloud name real uploadPreset: "cocina_upload" // ← nome do preset criado };
✅ Com isso, quando você fizer upload de foto de produto ou logo no painel admin, a imagem vai direto para o Cloudinary e a URL fica salva no Firestore. Rápido e sem custo no início!
7

Deploy — GitHub + Vercel

Colocar o site no ar com domínio personalizado

📁

Criar repositório no GitHub

Acesse github.com → "New repository". Nome: cocina-brasilena. Deixe Public (necessário para o Vercel gratuito). Clique em "Create repository".

📤

Fazer upload dos arquivos

Na página do repositório clique em "uploading an existing file". Arraste todos os arquivos do ZIP (descompactados) para lá. Clique em "Commit changes".

⚠️ Certifique-se de que o firebase-config.js já tem as credenciais reais antes de subir para o GitHub. O arquivo img/logo.png também deve estar incluído.
🚀

Deploy no Vercel

Acesse vercel.com → faça login com o GitHub. Clique em "New Project" → selecione o repositório cocina-brasilena. Clique em "Deploy" — sem precisar configurar nada!

🌐

Domínio personalizado (opcional)

No painel Vercel, vá em Settings → Domains. Adicione seu domínio. Siga as instruções de DNS que o Vercel mostra (normalmente configura em < 5 minutos).

🔒

Autorizar domínio no Firebase

Volte ao Firebase → Authentication → Settings → Domínios autorizados. Adicione o domínio do Vercel (ex: cocina-brasilena.vercel.app) e também o domínio personalizado, se tiver.

🎉 A cada vez que você fizer alterações e enviar para o GitHub, o Vercel atualiza o site automaticamente em segundos!
