🛠️ Problemas e Soluções

Guia rápido para resolver login, Cloudinary e regras do Firebase

🔐

Problema: Não consigo fazer login no Admin

Acesse: seusite.com/admin — deve aparecer a tela de login

✅ O que verificar:

1
Domínio autorizado no Firebase
Vá em Firebase Console → Authentication → Settings → Domínios autorizados
Adicione: localhost e o domínio do Vercel (ex: cocina-brasilena.vercel.app)
Se tiver domínio próprio, adicione ele também.
2
Usuário criado no Firebase
Firebase Console → Authentication → Users
Deve ter pelo menos 1 usuário na lista. Se não tiver, clique em "Add user" e crie com seu e-mail e senha.
3
E-mail verificado?
Não precisa verificar — o Firebase permite login mesmo sem verificação de e-mail.
4
Abrir o console do navegador (F12 → Console)
Se aparecer erro começando com auth/, a descrição exata do erro aparece na tela de login agora.
Erros comuns: auth/invalid-credential = senha errada · auth/network-request-failed = sem internet · auth/too-many-requests = aguardar alguns minutos
ℹ️ O endereço do painel admin é sempre: seusite.com/admin/index.html ou seusite.com/admin
📋

Regras do Firestore — Cole no Console

Firebase Console → Firestore Database → Aba "Regras"

Apague tudo que está no editor e cole exatamente este código:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Produtos: qualquer um lê, só admin escreve match /products/{productId} { allow read: if true; allow write: if request.auth != null; } // Slides do banner match /slides/{slideId} { allow read: if true; allow write: if request.auth != null; } // Configurações (logo, status) match /config/{configId} { allow read: if true; allow write: if request.auth != null; } } }

Depois clique em "Publicar". As regras ficam ativas em segundos.

⚠️ Se o projeto está em modo produção com as regras padrão (allow read, write: if false), nada funciona — nem leitura de produtos. Precisa publicar as regras acima.

Cloudinary — Como liberar o upload

Passo a passo para criar o preset não-assinado

ℹ️ O upload de imagens pelo navegador (sem backend) exige um Upload Preset "Unsigned". Sem isso, qualquer tentativa de upload retorna erro 401.
1
Acesse cloudinary.com → faça login → vá em Settings (ícone ⚙️ no topo)
2
Clique na aba "Upload"
3
Role a página até "Upload presets" → clique em "Add upload preset"
4
Na opção "Signing Mode", selecione "Unsigned" ← isso é obrigatório!
5
Anote o Preset name que aparece (ex: ml_default ou qualquer nome)
Pode deixar as outras opções padrão e clicar em "Save"
6
No Dashboard do Cloudinary, copie o Cloud name (ex: dxyz123abc)
7
Abra o arquivo js/firebase-config.js e preencha:
export const CLOUDINARY_CONFIG = { cloudName: "dxyz123abc", // ← seu Cloud name uploadPreset: "ml_default" // ← nome do preset criado };
✅ Com isso, ao fazer upload de foto no painel admin, a imagem vai direto para o Cloudinary e a URL é salva no Firestore automaticamente.

⚠️ CORS — se o upload falhar com erro de rede:

Vá em Cloudinary → Settings → Security → em "Allowed fetch domains" adicione o domínio do seu site (ex: cocina-brasilena.vercel.app). Se o campo estiver vazio, qualquer domínio é permitido — deixe vazio para simplificar.

🌐

Autorizar domínio no Firebase (obrigatório!)

Sem isso o login não funciona mesmo com e-mail e senha corretos

1
Firebase Console → Authentication
2
Clique na aba "Settings" (ao lado de "Users")
3
Em "Authorized domains" clique em "Add domain"
4
Adicione cada um desses domínios separadamente:
  • localhost — para testar localmente
  • cocina-brasilena.vercel.app — domínio do Vercel (substitua pelo seu)
  • seudominio.com — se tiver domínio próprio
⚠️ localhost geralmente já vem autorizado. Mas o domínio do Vercel você precisa adicionar manualmente toda vez que fizer deploy em um novo projeto.

Checklist — Tudo pronto para funcionar

Marque cada item antes de testar

  • ☐ Regras do Firestore publicadas (não mais allow read, write: if false)
  • ☐ Usuário admin criado em Authentication → Users
  • ☐ Domínio do Vercel autorizado em Authentication → Settings → Authorized domains
  • ☐ Cloudinary: preset "Unsigned" criado
  • ☐ firebase-config.js com cloudName e uploadPreset corretos
  • ☐ Arquivos atualizados no GitHub (versão v6)
  • ☐ Testei login em seusite.com/admin — entrou com sucesso
